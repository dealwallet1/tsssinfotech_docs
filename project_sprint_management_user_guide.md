# Project Sprint Management Module — Detailed User Guide

---

## 1. Overview

This document explains the complete workflow of the **Project Sprint Management Module**.

The purpose of this guide is to help users understand how to create projects, manage tasks, assign users, handle comments, upload attachments, and track project workflow without requiring additional guidance.

---

## 2. Create New Project

### Steps

1. Open the **Project Module**  
2. Click **New**  
3. Enter **Project Name**  

Example:

```text
aiml
```

4. Click **Save**  

### Result

The project is created and displayed in the Projects page.

---

## 3. Open Project

### Steps

1. Open the created project from the Projects page  
2. View project dashboard and task management options  

### Result

Project dashboard becomes available for managing tasks.

---

## 4. Create New Task

### Steps

1. Click **New Task**  
2. Enter Task Name  

Example:

```text
working on sales module
```

3. Continue with task details  

### Result

Task form opens for entering project details.

---

## 5. Fill Task Details

Inside the task form users can provide:

- Sprint  
- Story Points  
- Assignees  
- Team Lead  
- Tags  
- Customer  
- Priority  
- Start Date  
- Target Date  
- Tester  
- Size  

Fill required information and continue.

---

## 6. Description Section

Below Additional Info section, open the **Description Tab**.

### Purpose

Add task description or work details if required.

### Example

- Development notes  
- Work explanation  
- Implementation details  

---

## 7. Subtasks Management

Open the **Sub-tasks Tab**.

If work needs to be divided into smaller tasks:

### Steps

1. Add Subtask  
2. Enter subtask details  
3. Assign users  
4. Add tags and details  
5. Save subtask  

### Purpose

Break complex work into smaller manageable tasks.

---

## 8. Extra Information

The **Extra Info Section** is optional.

### Purpose

Additional project information can be stored if required.

No mandatory changes required unless project needs extra details.

---

## 9. Comments and Mentions

Open the **Comments Tab**.

At the bottom comment box:

### Steps

1. Type comment  
2. Mention user using `@` symbol  

Example:

```text
@indu
```

3. Dropdown list appears  
4. Select user  
5. Click **Comment** button  

### Result

Mentioned user receives notification.

---

## 10. Attachments Management

Inside attachments section users can:

- Upload screenshots  
- Upload images  
- Upload PDF documents  
- Upload project files  
- Preview attachments  
- Download attachments  
- Edit or delete attachments if required  

### Supported Files

PNG, JPG, PDF, DOC, DOCX, XLSX, PPT and other supported Odoo attachment formats.

---

## 11. Save Task

After entering all required information:

### Steps

1. Review task details  
2. Click **Save**  
3. Task becomes visible in Task Board  

### Result

Task is successfully created.

---

## 12. Task Workflow Tracking

Once task is created, task moves through workflow stages.

### Workflow Stages

```text
To Do → In Progress → Code Review → Testing → Done
```

### Purpose

Track task progress from creation to completion.

---

## 13. Complete Workflow Summary

The complete project workflow follows:

```text
Create Project
      ↓
Open Project
      ↓
Create Task
      ↓
Add Task Details
      ↓
Assign Users
      ↓
Add Description
      ↓
Create Subtasks
      ↓
Comments & Mentions
      ↓
Upload Attachments
      ↓
Save Task
      ↓
Track Workflow Stages
```

---

## 14. Benefits

This module provides:

- Better project organization  
- Sprint based task management  
- User assignment workflow  
- Team collaboration through comments  
- Attachment sharing support  
- Structured workflow tracking  
- Professional project management system  

---

## 15. Conclusion

The Project Sprint Management Module provides a complete project workflow management system for handling projects, tasks, sprints, collaboration, and task tracking.

By combining sprint management, task assignment, comments, attachments, and workflow tracking, the module creates a structured and efficient project management environment similar to professional tools like Jira.
