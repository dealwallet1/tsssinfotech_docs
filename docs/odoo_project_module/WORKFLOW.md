# Workflow

## Overview
This workflow follows a Jira-style Agile process implemented in the Odoo 19 Project module.  
It ensures a structured task lifecycle from creation to completion.

---

## Stages

1. **Backlog**
   - Task is created but not yet planned

2. **To Do**
   - Task is ready to start

3. **In Progress**
   - Developer is actively working on the task

4. **Code Review**
   - Reviewer verifies the implementation and code quality

5. **Testing**
   - Tester validates functionality and checks for issues

6. **Done**
   - Task is completed and verified

---

## Workflow Flow

Developer → Reviewer → Tester → Done

- Developer completes implementation  
- Reviewer checks and approves  
- Tester validates functionality  
- Task is marked as Done  

---

## Rejection Flow

Changes Requested → In Progress

- If reviewer or tester finds issues  
- Task is sent back to developer  
- Developer reworks and resubmits  

---

## Notes

- Each stage represents a clear responsibility in the team  
- Workflow ensures quality control before completion  
- Reduces errors and improves collaboration
